# App 整体 UI/UX 优化方案

更新时间：2026-05-06

## 状态

- 状态：active
- 说明：统一设备平台 UI/UX 的当前优化方向，具体页面落地以后再逐项归档。

## 1. 目标

当前 App 已经从单一消防吊桶 App 逐步重构为统一设备平台，产品包括：

- 消防吊桶
- 光伏清洗

下一阶段 UI/UX 目标不是单点修补，而是建立一套统一的 App 视觉和交互体系，让登录、首页、设备列表、控制台、设置页、悬浮窗看起来像同一个产品。

## 2. 设计定位

产品类型：工业无人机设备管理 App。

用户场景：

- 客户快速查看自己绑定的设备。
- 进入对应产品线控制设备。
- 在其他页面通过悬浮窗做快捷控制。
- 查看版本、OTA、悬浮窗权限和透明度等设置。

推荐风格：

- 现代 SaaS 企业风格。
- 浅色为主，背景 `#F5F7FA / #F8FAFC`。
- 设备卡片可使用高质量实拍图，但文字区域必须有足够遮罩。
- 控制面板采用白色或轻玻璃面板，避免厚重和过度说明。
- 功能色明确：蓝色主操作、绿色在线/成功、红色离线/失败、橙色警告。

不推荐：

- 默认 Material 紫色。
- 每个页面单独写一套颜色。
- 大段说明文字。
- 透明度过高导致文字不可读。
- 控制按钮过大、滑条过长、界面像调试工具。

## 3. 当前主要问题

### 3.1 主题系统不统一

当前 `ui/theme` 仍保留默认 Material 紫色：

- `Purple40`
- `Purple80`
- `Pink40`
- `dynamicColor = true`

问题：

- Android 12+ 动态色会让不同设备上主色不一致。
- App 实际页面使用蓝色、绿色、黄色、消防绿色等多套颜色。
- 登录页使用 `LoginColors.Primary = #6366F1`，主界面使用 `#1677FF`，主题层仍是紫色。

建议：

- 关闭动态色或至少默认关闭。
- 建立 `TjiColorTokens`。
- 全 App 主色统一为 `#1677FF`。
- 消防吊桶和光伏清洗只作为产品强调色，不作为全局主色。

### 3.2 字体体系不完整

当前 `Type.kt` 只覆盖了 `bodyLarge`，其他标题、按钮、标签仍大量依赖 Material 默认值或局部 `fontSize`。

问题：

- 页面标题、卡片标题、标签、数据值风格不统一。
- 有些地方字号偏大，有些地方过小。
- 控制台数据区和首页卡片的视觉层级不稳定。

建议字体体系：

| 用途 | 字号 | 字重 | 用法 |
| --- | --- | --- | --- |
| 页面标题 | 24-30sp | 700/800 | 首页、产品页 |
| 顶栏标题 | 18-20sp | 700 | 设备详情页 |
| 卡片标题 | 15-16sp | 600/700 | 控制卡、设置卡 |
| 正文 | 13-14sp | 400/500 | 简短说明 |
| 标签 | 11-12sp | 500/600 | 状态、字段名 |
| 数据值 | 18-24sp | 700/800 | 飞行状态、统计 |

字体建议：

- Android 原生可先用 `FontFamily.SansSerif` 做统一。
- 如果后续要更强品牌感，可接入 Plus Jakarta Sans 或类似企业 SaaS 字体。

### 3.3 页面风格不一致

当前不同页面的视觉语言：

- 登录页：蓝紫渐变、玻璃/白卡片。
- 首页：现代 SaaS 设备卡片，有实拍图。
- 光伏清洗控制页：白卡片 + 蓝色主控。
- 消防吊桶详情页：较简化，仍偏旧结构。
- 设置页：底部 Sheet，风格相对干净。
- 悬浮窗：半透明/全透明快捷控制器。

问题：

- 登录页和首页的主色、卡片圆角、阴影层级不完全一致。
- FireBucket 和 SolarClean 控制台视觉差距较大。
- 设置页是新风格，但一些字段文案还偏功能配置。

建议：

- 全局统一背景、卡片、圆角、阴影、状态色。
- 每个产品只在图标、图片、产品强调色上区别。
- 所有控制页采用同一套 `ControlCard / StatusCard / SettingsCard` 结构。

## 4. 全局设计系统建议

### 4.1 颜色 Token

建议建立统一 Token：

| Token | 色值 | 用途 |
| --- | --- | --- |
| `AppBackground` | `#F5F7FA` | 全局页面背景 |
| `Surface` | `#FFFFFF` | 主卡片 |
| `SurfaceSoft` | `#FAFBFF` | 卡片渐变尾色 |
| `PrimaryBlue` | `#1677FF` | 主操作、滑条、链接 |
| `TextPrimary` | `#1A1A2E` | 标题和主文字 |
| `TextSecondary` | `#667085` | 正文说明 |
| `TextMuted` | `#8C8C8C` | 标签和弱信息 |
| `OnlineGreen` | `#52C41A` | 在线、成功 |
| `WarningOrange` | `#FA8C16` | 警告、关闭类操作 |
| `ErrorRed` | `#FF4D4F` | 离线、失败、危险操作 |
| `BorderSoft` | `#E8EDF5` | 分割线和边框 |

### 4.2 圆角和阴影

| 组件 | 圆角 | 阴影 |
| --- | --- | --- |
| 页面主卡片 | 20-24dp | 轻阴影 |
| 控制卡片 | 12-16dp | 2-6dp |
| 小胶囊状态 | 999dp | 无 |
| 悬浮窗 | 12dp | 轻阴影或透明 |
| 输入框 | 14-16dp | 无或极轻 |

注意：

- 当前 SolarClean 控制台要求 `12px`，Compose 中可用 `12.dp`。
- 首页产品卡可以更大圆角，但控制台要更紧凑。

### 4.3 动效规范

所有微交互动效：

- 时长：150-220ms。
- 曲线：`FastOutSlowInEasing` 或 spring。
- 避免动画宽高变化。
- 优先动画：`offset`、`alpha`、`color`、`scale`。

必须修复：

- 悬浮窗 `MiniSwitch` 不能用 `contentAlignment` 硬切位置。
- 应改为 `animateDpAsState` 或 `animateFloatAsState` 控制 thumb offset。

## 5. 分页面优化建议

### 5.1 登录页

当前问题：

- 登录页视觉和后续首页仍有一点割裂。
- 输入框缺少显式 label，只靠 placeholder。
- 记住密码逻辑已经修过，但 UI 上“记住我/忘记密码”比较传统。
- `LoginWidget.kt` 有大量注释旧代码，影响维护。

建议：

- 保留当前现代登录卡方向。
- 背景与首页统一为浅蓝灰渐变。
- 登录卡片用 `Surface` 或轻玻璃，不建议全透明。
- 输入框增加稳定 label 或顶部小标题。
- 删除大段废弃注释代码。
- 登录按钮使用全局 `PrimaryBlue`，不要继续紫色。

优先级：P1。

### 5.2 平台首页

当前优点：

- 已经接近“我的设备 / 产品概览”风格。
- 产品卡片有实拍图，方向正确。
- 横屏两列布局已处理过。

当前问题：

- 首页、产品入口、设备列表之间层级还可以更清晰。
- 图标、卡片、标题的 tokens 仍是局部 hardcode。
- 空状态要更像客户页面，而不是调试提示。

建议：

- 首页只保留用户会关心的信息：我的设备、在线设备、设备类型。
- 产品卡片继续使用图像右侧 + 左侧渐变遮罩。
- 所有空状态文案保持短句：`暂无设备`、`请添加或联系管理员开通`。
- 设置入口固定右上角，保持可见。

优先级：P1。

### 5.3 产品设备列表页

当前问题：

- 设备卡片还可以进一步统一 FireBucket 和 SolarClean。
- 在线/离线是文字 + 圆点，方向正确，但布局有时拥挤。
- 设备名和 `deviceId` 相同时应只显示一行。

建议：

- 卡片结构统一为：图标 + 设备名 + 状态 + 箭头，`deviceId` 只在不同于设备名时显示。
- 状态放右侧，颜色 + 文字，不只靠颜色。
- 点击整张卡片进入控制台。

优先级：P0/P1。

### 5.4 光伏清洗控制页

当前优点：

- 控制卡在最上方，飞行状态在下方，符合操作优先。
- 控制和飞行状态支持展开/收起并记忆状态。
- ACK 已转为用户可读文案。
- 设置页包含 OTA，符合方案 A。

当前问题：

- 控制卡和飞行状态卡局部仍有硬编码字体和颜色。
- 离线控制是否禁用需要统一。
- 数据卡片在小屏和横屏下还需要持续检查。

建议：

- 离线时禁用控制按钮、滑条和悬浮窗控制。
- ACK 胶囊抽成公共组件。
- `ControlSlider` 和悬浮窗 `FloatingControlSlider` 抽象出共享视觉 token。
- 飞行状态里的数值可考虑按重要度分组，不要所有字段同权重。

优先级：P0/P1。

### 5.5 消防吊桶控制页

当前问题：

- 相比 SolarClean，FireBucket 控制页仍偏旧。
- Link 与桶的层级容易让用户困惑。
- 当前水桶特殊逻辑：客户有多个 Link，一个 Link 下多个桶，最终控制桶但 MQTT 发给 Link。UI 需要解释这种层级，但不要写成大段说明。

建议：

- 页面标题显示 Link 名称。
- 第一层卡片展示 Link 状态。
- 第二层展示桶列表，每个桶卡片直接可控制。
- 不再出现重复两栏或重复状态。
- 消防吊桶也统一卡片圆角、阴影、状态胶囊。

优先级：P1。

### 5.6 设置页

当前优点：

- 底部 Sheet 形式合理。
- 已有悬浮窗开关、权限、透明度、版本号。

当前问题：

- “悬浮窗透明度”从用户角度可能不如“悬浮窗背景强度”好理解。
- 横屏时底部内容可能被系统栏或高度限制影响。

建议：

- 文案改为“悬浮窗背景强度”，显示 `0% = 全透明`。
- Sheet 最大高度根据横屏适配，可用 `fillMaxHeight(0.92f)` 或保留滚动但确保顶部/底部都能到达。
- 设置项分组：悬浮窗、账号、版本。

优先级：P1。

### 5.7 悬浮窗

当前优点：

- 已经紧凑化。
- 默认居中。
- 支持透明度设置。
- 显示当前进入的设备。
- ACK 在标题栏反馈。

当前问题：

- Switch thumb 动画不顺滑，会短暂方块/跳变。
- 离线时仍可能可点击，需要禁用。
- 全透明时文字可读性依赖背景。

建议：

- 修复 `MiniSwitch` 动画。
- 离线禁用控制。
- 保留透明度设置，但建议用户默认可调。
- 标题栏信息优先级：产品图标、`deviceId`、在线状态、ACK、最小化、关闭。

优先级：P0。

## 6. 代码结构建议

### 6.1 建立 UI Tokens

建议新增：

```text
app/src/main/java/com/tji/device/ui/design/
├── TjiColors.kt
├── TjiTypography.kt
├── TjiShape.kt
├── TjiElevation.kt
└── TjiMotion.kt
```

这样避免每个页面散落：

- `Color(0xFF1677FF)`
- `RoundedCornerShape(12.dp)`
- `fontSize = 13.sp`
- `CardDefaults.cardElevation(...)`

### 6.2 建立公共组件

建议抽出：

```text
ui/components/
├── TjiStatusBadge.kt
├── TjiCommandFeedbackBadge.kt
├── TjiControlSlider.kt
├── TjiMiniSwitch.kt
├── TjiSettingRow.kt
└── TjiDeviceCard.kt
```

收益：

- 主页面和悬浮窗 ACK 胶囊统一。
- SolarClean 和 FireBucket 状态展示统一。
- Switch 动画修一次，所有地方受益。

### 6.3 Preview 要求

所有重要 UI 必须有 Preview：

- 登录页。
- 首页。
- 产品设备列表页。
- FireBucket 控制页。
- SolarClean 控制页。
- SolarClean 设置页。
- 悬浮窗展开/未连接/透明背景。
- 关键图标预览。

Preview 必须覆盖：

- 在线。
- 离线。
- 空设备。
- 横屏。
- 小屏宽度。

### P1：近期优化

- 统一主题色，去掉默认紫色主题。
- 建立全局字体层级。
- 登录页主色改为平台蓝。
- 设置页“透明度”文案优化为“背景强度”。
- 消防吊桶控制页同步现代卡片风格。
- 抽公共 `CommandFeedbackBadge`。

### P2：后续设计系统化

- 已完成：抽出基础 Design Token，并把主色、背景、文字、状态色统一到 `ui/theme`。
- 已完成：抽出公共卡片、按钮、滑条、MiniSwitch、ACK 反馈徽标、在线状态组件。
- 已完成：首页、产品页、光伏控制页、消防吊桶控制页、悬浮窗统一接入公共 UI 组件。

- 后续持续项：少量图标预览和特殊场景仍可继续补充，但不阻塞当前 P2 骨架。

## 8. 验收标准

整体 App UI 优化完成后，应满足：

- 登录、首页、产品页、控制页、设置页看起来属于同一个 App。
- 全局主色不再混用紫色和蓝色。
- 页面标题、卡片标题、正文、标签、数据值有固定字号和字重。
- 所有客户能点的区域至少 44dp，或视觉小但触控区域足够。
- 离线状态下不会发送控制命令。
- ACK 反馈只显示用户能理解的结果。
- 悬浮窗 Switch 切换丝滑，没有方块跳变。
- 横屏、小屏下无明显内容遮挡或大片空白。
