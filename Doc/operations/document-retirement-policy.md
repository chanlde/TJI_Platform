# 文档删除与归档规则

## 状态

- 状态：active
- 说明：用于判断文档是否可以删除、归档或继续保留。

## 适用范围

本规则适用于 `Doc/` 下的业务正文文档。目录 README、ADR 和迁移表按各自用途维护，不需要按本文流程删除。

## 生命周期

- `active`：当前仍作为实现、联调、产品或测试依据使用。
- `draft`：仍在讨论或联调中，不能删除。
- `deprecated`：确认不再作为当前依据，但删除前必须保留替代位置和删除条件。

## 标记 Deprecated 的条件

只有同时满足以下条件，才能把业务文档改为 `deprecated`：

- 对应功能、协议或架构规则已被新文档、代码实现或外部系统替代。
- 当前文档不再被 README、产品线索引、迁移表以外的正文引用。
- 相关产品、硬件、服务端或测试负责人已经确认不再依赖该文档。

## Deprecated 文档必须补充

标记为 `deprecated` 的文档必须新增：

```text
## 废弃说明

- 替代文档：...
- 删除条件：...
```

如果没有替代文档，写明“无替代文档”并说明原因。删除条件应具体到可验证状态，例如“所有历史链接已迁移”“相关接口已下线”“对应产品线已移除”。

## 删除步骤

1. 将文档状态改为 `deprecated`，补齐废弃说明。
2. 运行 `./gradlew checkDocs`，确认没有旧路径、断链和索引遗漏。
3. 在迁移表或相关 README 中确认是否需要保留历史入口。
4. 删除文件后再次运行 `./gradlew checkDocs` 和 `./gradlew check`。

如果任何一步无法确认，先保留文档，并把问题记录到 [review-questions.md](review-questions.md)。
